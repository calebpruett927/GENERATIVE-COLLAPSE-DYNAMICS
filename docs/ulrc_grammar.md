## Overview

ULRC preserves the structural meaning of the corpus's terms — collapse, return, contract,
closure, trace, missingness, ledger, stance, integrity, seam, weld, admissibility, and
domain readback — as they travel across domains, pedagogy, public explanation, and
translation. It is not terminology management or decoration; it is a **structural
necessity** for a field whose object must return across domains without losing its
admissibility conditions.

**Governing rule.** *Translate without drift.*
**Final burden.** Make meaning **return**.

## Language as Return

ULRC treats language under the same collapse–return discipline as claims and closures:

- A word must return through translation.
- A phrase must return through critique.
- A symbol must return through formalization.
- A domain term must return through cross-domain mapping.

If a term's burden is lost, it has *repeated* but not *returned*. Thus **ULRC is the
linguistic form of Reditus**: it asks whether meaning itself returns.

## Structural Term and Structural Burden

A **structural term** is a word, phrase, symbol, or expression whose meaning is determined
by the **burden** it carries inside the GCD architecture. A term is preserved only when it
continues to perform the same structural work — not when it merely keeps its spelling.

| Term | Structural burden (may **not** be reduced to…) |
|------|------------------------------------------------|
| **Return** | Re-entry under declared conditions after collapse — not repetition, resemblance, improvement, or ordinary recovery. |
| **Collapse** | A pressure/perturbation/rupture event that tests whether structure can return — not destruction alone. |
| **Closure** | The declared rule that decides passage, failure, return, non-return, or non-evaluability — not a conclusion. |
| **Contract** | The frozen rule-set under which evaluation becomes admissible — not theme, intention, or informal expectation. |
| **Integrity** | A derived read after reconciliation — not asserted before the ledger; not moral approval. |
| **Real** | Admissible after collapse–return under declared structure — not subjective intensity, persuasion, or loose metaphysics. |
| **Stance** | A derived verdict under contract — not opinion, preference, or premature conclusion. |

## Formal Translation Rule

Let `T` be a structural term and `M(T)` its translation into another register, domain, or
audience. A translation is admissible only when the operative burden is invariant:

```
Burden(M(T)) = Burden(T)
```

If the wording changes but the burden is invariant, the term has **returned** through
translation. If the wording stays similar but the burden changes, the term has **repeated**,
not returned. *Words may change; burden may not.*

## The Rosetta — Cross-Domain Translation

ULRC maps the five words (Drift · Fidelity · Roughness · Return · Integrity) across lenses so
domains can read each other in their own dialect without losing auditability. Integrity is
**never asserted** in the Rosetta — it is read from the reconciled ledger.

| Lens | Drift | Fidelity | Roughness | Return |
|------|-------|----------|-----------|--------|
| **Epistemology** | Change in belief/evidence | Retained warrant | Inference friction | Justified re-entry |
| **Ontology** | State transition | Conserved properties | Heterogeneity / seams | Restored coherence |
| **Phenomenology** | Perceived shift | Stable features | Distress / bias / effort | Coping / repair that holds |
| **History** | Periodization | Continuity | Rupture / confound | Reconciliation |
| **Policy** | Regime shift | Mandate persistence | Friction / cost | Reinstatement |
| **Semiotics** | Sign drift | Ground persistence | Translation friction | Interpretant closure |

The columns' meanings stay stable while the dialect changes. This is what makes cross-domain
synthesis mechanical rather than interpretive — the prose maps through the chosen lens, while
the kernel relation IC = exp(κ) provides unitless multiplicative comparability across seams.

## Tier Relation

ULRC is **cross-tier** and is **not a tier**:

| Level | ULRC function |
|-------|---------------|
| **Tier-1** | Stabilizes kernel vocabulary; prevents symbolic drift of reserved symbols. |
| **Tier-0** | Names protocol operations; protects the evaluative sequence's language. |
| **Tier-2** | Translates domain readbacks into shared structural language without redefining the kernel. |

> **ULRC succeeds when meaning returns through translation.** It fails when return becomes
> repetition, closure becomes conclusion, stance becomes opinion, or integrity is asserted
> before the ledger — *repetition without return of meaning*.
