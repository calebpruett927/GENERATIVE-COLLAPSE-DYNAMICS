## I. Prooemium — Why Latin

> *Lingua Latina non ornamentum est, sed sutura inter mathematicam, logicam, et linguisticam.*
>
> ("Latin is not ornament; it is the seam between mathematics, logic, and linguistics.")

Latin serves three structural functions in this system that no other language provides simultaneously:

1. **Morphological precision.** Latin inflection carries operational constraints that English paraphrase loses. Saying *derivatio* (diversion from a channel) produces different — and more constrained — reasoning than saying "drift." The morphology IS the specification.

2. **Namespace isolation.** Every natural language carries pre-loaded associations. "Entropy" in English collides with thermodynamics, information theory, disorder, chaos, colloquial "things fall apart." *Entropia campi Bernoulliani* collides with nothing. Latin terms function as reserved identifiers in a namespace that no living field actively occupies.

3. **Cross-domain bridge.** The Rosetta adapter translates between epistemology, ontology, phenomenology, history, and policy using five words. The Latin seeds (derivatio, fidelitas, curvatura, reditus, integritas) are the *canonical names* — the pre-translation layer from which all lens-specific dialects derive. The Latin does not privilege any one lens. It sits beneath all of them.

This document formalizes the complete Latin vocabulary, operational verbs, typed grammatical patterns, and governing maxims of the system. It is the **surface language** that translates between the barriers of mathematics, logic, and linguistics — *tres muros* that no single notation can bridge alone.

- **Fidelitas** (f. 3rd decl.) — from *fides* (trust, faith). The -itas suffix marks an abstract quality: "the state of being faithful." What survives is what kept faith with the channel.

## V. Septem Verba Operativa — The Seven Imperative Verbs

> *Imperativi sunt operationes; coniunctivi sunt quaerentes.* — Imperatives are operations; subjunctives are seekers.

Every computable operation in the system maps to exactly one of seven Latin imperative verbs. These are the complete API of the discourse layer.

| # | Verbum | Imperative | Operation | Maps To |
|---|--------|------------|-----------|---------|
| 1 | **Dicere** | *Dic* | Compute and report | `kernel_optimized.compute_invariants()`, any read of F, ω, S, C, κ, IC |
| 2 | **Reconciliare** | *Reconcilia* | Balance the budget | `seam_optimized.compute_seam_budget()`: Δκ = R·τ_R − (D_ω + D_C) |
| 3 | **Suturare** | *Suturā* | Weld / validate across the seam | `validator.validate()`, `cli._cmd_validate()`: schema + identity + regime + SHA-256 |
| 4 | **Comparare** | *Comparā* | Pairwise comparison | Rosetta translation: compare two domains through the five words |
| 5 | **Ligare** | *Liga* | Bind to contract | `frozen_contract.FrozenContract()`: freeze all parameters for a run |
| 6 | **Inscribere** | *Inscribe* | Append to ledger | `ledger/return_log.csv`: append-only record. *Historia numquam rescribitur.* |
| 7 | **Verificare** | *Verifica* | Prove identity | Tier-1 identity checks: F + ω = 1, IC ≤ F, IC ≈ exp(κ) |

### Verb Semantics

- **Dic** is the workhorse — any computation that produces a value. "Say what you found."
- **Reconcilia** is bookkeeping — debits and credits must balance within tolerance.
- **Suturā** is the full validation pass — the deepest operation, crossing the seam. The accent on -ā is deliberate: first conjugation imperative.
- **Comparā** invokes the Rosetta. It requires two operands and a lens.
- **Liga** freezes. Once called, parameters are immutable for the run.
- **Inscribe** is append-only. Once written, never erased. *Sutura tantum additur.*
- **Verifica** is the identity proof — the simplest and most fundamental operation. Does 1 = 1? Does F + ω = 1?

### Composition

Verbs compose left-to-right in execution order:

```
Liga → Dic → Reconcilia → Suturā → Inscribe
```

This is the standard validation pipeline: bind contract → compute kernel → balance budget → weld → write to ledger. **Comparā** and **Verifica** are callable at any point as diagnostic side-channels.

## VI. Formulae Latinae — Eight Grammatical Patterns as Types

> *Grammatica est typus; declinatio est signum operationis.*
>
> ("Grammar IS the type system; declension IS the operation marker.")

Latin grammatical structure encodes computation type. The case system, mood system, and clause structure of Latin map directly to different categories of operation. Eight patterns are defined, each with a fixed grammatical form and a fixed computational meaning.

### Pattern 1: NOM + GEN — Read / Query

**Grammar:** Nominative noun + genitive qualifier.
**Meaning:** Report the value of the specified quantity.

```
Fidelitas Ferri.
```
> "The fidelity of Iron." → Return F for iron's 8-channel trace.

```
Entropia Nuclei Uranici.
```
> "The entropy of the uranium nucleus." → Return S for uranium's kernel.

**Type signature:** `(Symbol, Domain) → Scalar`

### Pattern 2: IMPER + ACC — Execute / Command

**Grammar:** Imperative verb + accusative object.
**Meaning:** Execute the operation on the given object.

```
Dic invariantes Hydrogenii.
```
> "Report the invariants of Hydrogen." → Compute full kernel (F, ω, S, C, κ, IC) for H.

```
Suturā casepackum financiae.
```
> "Weld the finance casepack." → Run full validation on the finance casepack.

```
Verifica identitatem primam.
```
> "Verify the first identity." → Check F + ω = 1.

**Type signature:** `Verb × Object → Result`

### Pattern 3: ACC + AD + ACC — Parameterize / Configure

**Grammar:** Accusative + *ad* + accusative (motion toward).
**Meaning:** Set a parameter or configure a scope.

```
Epsilon ad 1e-8.
```
> "Epsilon to 10⁻⁸." → Set guard band. (*Trans suturam congelatum* — this is the ONLY admissible value.)

```
Tolerantiam ad 0.005.
```
> "Tolerance to 0.005." → Set seam tolerance tol_seam.

**Type signature:** `Parameter × Value → Configuration`
**Constraint:** Parameters are frozen. This pattern documents; it does not invent.

### Pattern 4: ABL.ABS. — Set Context / Scope

**Grammar:** Ablative absolute (noun + participle, both ablative).
**Meaning:** Establish the context within which subsequent operations execute.

```
Contractu congelato, dic invariantes.
```
> "With the contract frozen, report invariants." → Freeze first, then compute.

```
Regimine collapsūs declarato, reconcilia rationem.
```
> "With collapse regime declared, reconcile the account." → After regime classification, run budget.

```
Suturā clausā, inscribe in rationarium.
```
> "With the seam closed, write to the ledger." → Only after validation passes.

**Type signature:** `Context × Operation → Scoped(Result)`
**Semantics:** The ablative absolute creates a *scope* — a frozen context that governs everything within its clause. This is isomorphic to `with contract.frozen():` in Python.

### Pattern 5: GEN + ET + GEN — Compare / Contrast

**Grammar:** Genitive + *et* + genitive (possession of two entities).
**Meaning:** Compare two entities through the Rosetta.

```
Quarki et Hadronis.
```
> "Of the quark and of the hadron." → Compare kernel signatures across the confinement boundary.

```
Ferri et Nickelī.
```
> "Of Iron and of Nickel." → Compare periodic kernel signatures at the stability valley.

```
Financiae et Astronomiae.
```
> "Of Finance and of Astronomy." → Cross-domain Rosetta comparison through the five words.

**Type signature:** `(Domain, Domain) → Comparison`
**Implicit verb:** *Comparā* is understood when two genitives are juxtaposed.

### Pattern 6: NONNE / NUM — Assert / Test

**Grammar:** *Nonne* (expects "yes") or *Num* (expects "no") + indicative clause.
**Meaning:** Assert a property. Nonne = should-pass; Num = should-fail.

```
Nonne F + ω = 1?
```
> "Surely F + ω = 1?" → Assert the duality identity. Expected: CONFORMANT.

```
Nonne IC ≤ F?
```
> "Surely IC ≤ F?" → Assert the integrity bound. Expected: CONFORMANT.

```
Num IC excedit F?
```
> "Surely IC does not exceed F?" → Assert violation. Expected: NONCONFORMANT (if IC > F is claimed).

**Type signature:** `Assertion → {CONFORMANT, NONCONFORMANT, NON_EVALUABLE}`
**Three-valued:** The answer is never boolean. *Tertia via semper patet.*

### Pattern 7: PARTICIPLE CHAIN — Pipeline / Sequential Composition

**Grammar:** Chain of perfect passive participles (ablative) governing a final main verb.
**Meaning:** Execute a sequence of operations, each feeding the next.

```
Contractu ligato, invariantibus dictis, ratione reconciliatā, suturā.
```
> "With the contract bound, with invariants reported, with the account reconciled, weld." → The full validation pipeline in one sentence.

```
Tractu normalizato, nucleo computato, regimine classificato, inscribe.
```
> "With the trace normalized, with the kernel computed, with the regime classified, write." → Normalize → compute → classify → log.

**Type signature:** `[Context₁, Context₂, ..., Contextₙ] → Operation → Result`
**Semantics:** Each participle is an ablative absolute that must complete before the next begins. This is a strict sequential pipeline with no parallelism. The final verb is the output operation.

### Pattern 8: UT + SUBJUNCTIVE — Threshold Seek / Goal-Directed

**Grammar:** *Ut* (in order that / so that) + subjunctive mood.
**Meaning:** Seek a parameter value or state that satisfies a condition.

```
Ut IC superet 0.30, quaere c minimum.
```
> "In order that IC exceed 0.30, seek the minimum c." → Find the threshold channel value where IC first exits Critical.

```
Ut regimen stabile sit, quaere ω maximum.
```
> "In order that the regime be stable, seek the maximum ω." → Find the drift ceiling for Stable (answer: ω < 0.038).

```
Ut sutura claudatur, quaere R minimum.
```
> "In order that the seam close, seek the minimum R." → Find R_crit for the current state.

**Type signature:** `Goal(Condition) → Search → Parameter`
**Semantics:** The subjunctive marks the clause as *desired, not actual*. This is a constrained optimization: find the value that makes the condition true. Not all goals are achievable — NON_EVALUABLE is a valid result.
